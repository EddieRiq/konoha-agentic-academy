# Yamanaka memory note template

This template is used to record mission memory in the Yamanaka Memory Network.

Memory notes preserve useful context, evidence, decisions, and follow-up work. They do not grant permission, expand mission scope, or replace source files.

## Frontmatter

```yaml
---
title: ""
memory_type: mission_note
status: draft
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
mission_id: ""
village: academy
visibility: public
sensitivity: low
source_files: []
related_doctrine: []
related_scrolls: []
tags: []
owner: ""
review_required: false
---
```

## Summary

Write a short summary of what this memory note captures.

Keep it factual. Do not turn uncertainty into conclusions.

## Source context

List the source material used to create this note.

```text
- Mission Charter:
- Mission Report:
- Files reviewed:
- Commands run:
- User confirmations:
- External sources:
```

## What happened

Describe the mission or event in chronological order.

Prefer concrete facts over interpretation.

## Evidence

List the evidence that supports this note.

```text
- File:
- Diff:
- Command output:
- Test result:
- User message:
- Review comment:
```

## Decisions recorded

Record decisions only if they were explicit.

```text
- Decision:
  - Made by:
  - Date:
  - Reason:
  - Scope:
```

Do not infer decisions from agent behavior.

## Useful context for future missions

Capture context that may help future work.

```text
- Relevant paths:
- Naming conventions:
- Known constraints:
- Risks:
- Open questions:
```

## What this note does not authorize

State limits clearly.

```text
This memory note does not authorize:
- file changes;
- command execution;
- doctrine updates;
- access to private context;
- external actions;
- mission completion.
```

## Sensitive content check

Before saving this note, confirm:

```text
- No secrets are included.
- No credentials are included.
- No private personal data is included.
- No local-only content is included unless stored inside a local Village.
- No copyrighted or private assets are embedded.
```

If sensitive content is required, keep the note in the appropriate local Village memory and mark sensitivity accordingly.

## Follow-up work

List follow-up work without treating it as approved scope.

```text
- Proposed follow-up:
  - Reason:
  - Requires approval: yes/no
  - Suggested owner:
```
