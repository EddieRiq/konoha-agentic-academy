# Learning proposal template

This template is used when an agent or human identifies a lesson that may improve future missions.

A learning proposal is not doctrine. It is a candidate lesson that requires review and approval before it can change Academy rules, Scrolls, templates, or behavior.

## Frontmatter

```yaml
---
title: ""
proposal_type: learning_proposal
status: draft
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
mission_id: ""
village: academy
visibility: public
sensitivity: low
proposed_by: ""
review_level: jounin
related_memory_notes: []
related_doctrine: []
related_scrolls: []
tags: []
---
```

Mission ID:

## Evidence

List concrete evidence.

```text
- What happened:
- What was expected:
- What failed or worked:
- Files involved:
- Commands or checks:
- Review comments:
```

Do not use vague confidence as evidence.

## Problem or opportunity

Explain the issue this proposal addresses.

```text
- Problem:
- Impact:
- Frequency:
- Risk if ignored:
```

## Proposed change

Describe the suggested change.

```text
- Change type:
  - Doctrine
  - Scroll
  - Template
  - Tool
  - Adapter
  - Memory practice
  - Review practice
  - Other

- Proposed target path:
- Proposed wording or behavior:
```

Do not modify the target file directly from this proposal.

## Scope

Define where this lesson applies.

```text
Applies to:
Does not apply to:
Local-only: yes/no
Public Academy candidate: yes/no
```

## Safety and privacy check

```text
- Does this proposal include sensitive content? yes/no
- Does it mention secrets or credentials? yes/no
- Does it expose private local Village context? yes/no
- Does it rely on copyrighted or private assets? yes/no
- Does it change permissions, approvals, memory, security, or doctrine? yes/no
```

If any answer increases risk, escalate review before adoption.

## Alternatives considered

```text
Alternative:
Reason not selected:
```

## Review requirement

Suggested review level:

```text
- Clerk review
- Jounin review
- Kage Summit review
- Human approval
```

Kage Summit review is required for changes that affect doctrine, safety, approval, memory, permissions, security, external actions, or cross-Village behavior.
