# Code of Conduct

## Purpose

Konoha Agentic Academy is an open-source project for building local-first agentic workflows, reusable Scrolls, safe orchestration patterns, and human-readable doctrine.

This Code of Conduct defines how human contributors are expected to behave when participating in the project.

Agent behavior is governed separately by:

- `core/conduct/AGENT_CONDUCT.md`
- `core/laws/KONOHA_LAWS.md`
- the policies under `protocols/`

## Core expectation

Contributors must act with respect, clarity, and accountability.

Disagreement is welcome. Harassment, personal attacks, intimidation, bad-faith behavior, and careless handling of private information are not.

## Expected behavior

Contributors are expected to:

- be respectful toward other contributors;
- explain technical disagreements with evidence;
- keep discussions focused on the project;
- ask clarifying questions before assuming intent;
- accept review and correction without hostility;
- give review feedback that is specific and actionable;
- disclose when a contribution was generated or heavily assisted by an AI agent;
- avoid submitting speculative claims as fact;
- avoid uploading secrets, private files, personal data, proprietary context, or copyrighted assets that are not license-safe.

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, insults, threats, discrimination, or personal attacks;
- publishing someone else's private information without consent;
- submitting credentials, tokens, `.env` files, private keys, or sensitive workplace data;
- knowingly submitting copyrighted or franchise-specific assets to the public repository without proper rights;
- pretending agent-generated work was fully human-written when that matters for review;
- fabricating evidence, benchmarks, security claims, or compatibility claims;
- repeatedly ignoring project safety rules, review requests, or maintainer guidance;
- using the project to hide malicious instructions, prompt injections, unsafe commands, or secret-exfiltration patterns.

## Privacy and sensitive information

Do not submit private or sensitive material to the public repository.

This includes, but is not limited to:

- credentials;
- tokens;
- `.env` files;
- private keys;
- internal company documents;
- personal data;
- private emails;
- customer data;
- local Village memory;
- local Village assets;
- copyrighted character assets, voices, logos, music, or franchise-specific designs.

If sensitive material is submitted by mistake, stop the discussion, notify maintainers, and avoid repeating the sensitive content in comments. If credentials were exposed, the owner should rotate them.

## AI-assisted contributions

AI-assisted contributions are allowed, but they must be reviewable.

When submitting AI-assisted work, contributors should:

- say which agent or model assisted when relevant;
- review the full diff before submitting;
- remove private or sensitive content;
- verify claims, links, commands, and examples;
- avoid submitting large unreviewed batches;
- include evidence for behavioral, safety, or compatibility changes.

AI assistance does not transfer responsibility away from the contributor.

## Disagreement and review

Technical disagreement should be handled through evidence and clear reasoning.

Good review comments explain:

- what the issue is;
- why it matters;
- what should change;
- whether the issue blocks merge or is only a suggestion.

Maintainers may ask contributors to split large pull requests, provide examples, add tests, remove unsafe content, or rewrite unclear doctrine.

## Enforcement

Maintainers may take action when this Code of Conduct is violated.

Possible actions include:

1. clarification or correction;
2. warning;
3. request to edit or remove content;
4. temporary restriction from discussion or contribution;
5. permanent ban from the project community;
6. reporting malicious or illegal activity to the relevant platform or authority when necessary.

Enforcement should be proportional to severity, risk, intent, and repetition.

## Reporting

If you see behavior that violates this Code of Conduct, report it to the project maintainers.

When reporting, include:

- a short description of what happened;
- links or screenshots if available;
- why it may violate this Code of Conduct;
- whether sensitive information is involved.

Do not publicly repost sensitive information while reporting an issue.

## Scope

This Code of Conduct applies to project spaces, including:

- GitHub issues;
- pull requests;
- discussions;
- documentation;
- community chats;
- public project events;
- any official Konoha Agentic Academy collaboration space.

It also applies when a contributor represents the project in public.

## Relationship to project doctrine

This file governs human community behavior.

Agent behavior is governed by `AGENT_CONDUCT.md` and the Konoha policies.

If there is a conflict between openness and safety, safety wins.
