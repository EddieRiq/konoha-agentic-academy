# Context acquisition and genuine missing context

Konoha must exhaust explicitly authorized local evidence before asking the human
for information.

Required order:

1. Load public doctrine and referenced local files from the current Git workspace.
2. Load registered agent families and capability policy.
3. Run deterministic local provider-readiness probes.
4. Load the read-only command policy.
5. Classify unresolved information.
6. Ask the human only for an unavailable source, a contradictory rule, a new
   permission, a budget decision or another genuine human choice.

`context_not_loaded` is not `user_input_required`.

The current Git workspace is authorized for public read-only inspection. Ignored,
private, secret-bearing and external paths remain excluded unless separately
approved.

Provider readiness is evidence discovered by deterministic probes. It is not a
question for the human.

A registered Jounin family performs independent review. It cannot implement,
review itself or approve instruction changes.

No assumptions still applies: local evidence may be read and verified, but it
must not be converted into permission or normative truth beyond its explicit
content.
