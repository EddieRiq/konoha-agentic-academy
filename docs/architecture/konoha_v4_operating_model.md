# Konoha 4.0 Operating Model

Codex conduce la conversación, comprende la misión y diseña el grafo operativo.
Hokage no inventa estrategia: valida constitucionalmente alcance, fuentes, permisos,
presupuesto y capacidades. Los workers son familias especializadas; sus salidas son
evidencia. Jounin revisa. Shikamaru propone cambios de instrucciones. Ningún modelo
aprueba su propio trabajo.

## Cero inferencia normativa

Un modelo puede razonar, pero no puede convertir una inferencia en hecho, regla,
requisito o autorización. Todo requisito debe provenir del usuario, una regla aprobada
o una fuente identificada. Ante ausencia, detenerse y preguntar.

## Aprobaciones

Una aprobación de plan cubre el grafo read-only hashado. Se vuelve a pedir permiso
ante mutación, red, contexto privado no cubierto, cambio de alcance, Git, cambio de
instrucciones o presupuesto excedido.
