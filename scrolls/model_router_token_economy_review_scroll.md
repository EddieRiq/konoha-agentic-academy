# Model Router and Token Economy Review Scroll

Use this Scroll to review model routing and token economy outputs.

## Review questions

1. Did Konoha mark model routing as evidence only?
2. Did it avoid invoking models in this layer?
3. Did it avoid downloading local models?
4. Did it distinguish actual token usage from estimated usage?
5. Did it avoid treating estimates as truth?
6. Did it recommend local models for private or low-risk tasks when appropriate?
7. Did it require stronger review for high-risk or critical tasks?
8. Did it preserve local/private boundaries?
9. Did it avoid shell, network, Git, repository apply, and background agents?
10. Did the token economy summary suggest useful optimization without rewriting doctrine?

## Non-authority rule

A model routing decision, token usage ledger, download plan, or economy summary does not authorize execution, model invocation, local model download, repository apply, Git operations, private context access, or mission closure.
