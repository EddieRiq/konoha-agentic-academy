# Yamanaka Advanced Memory Review Scroll

## Review purpose

Use this Scroll to review advanced Yamanaka memory and context pack outputs.

## Required checks

1. Confirm memory root is explicit.
2. Confirm Mission Workspace path is explicit.
3. Confirm memory capture uses the exact token `RECORD_YAMANAKA_MEMORY`.
4. Confirm context pack build uses the exact token `BUILD_CONTEXT_PACK`.
5. Confirm generated memory notes are Obsidian-compatible Markdown.
6. Confirm context packs include references and boundaries.
7. Confirm indexes do not include private paths accidentally.
8. Confirm no model invocation occurred.
9. Confirm no adapter invocation occurred.
10. Confirm no repository apply, staging, commit, push, or doctrine rewrite occurred.

## Non-authority rule

Memory supports action but does not authorize action.

Context packs are not permission.

Summaries are not truth.

A memory note or context pack must not be treated as a mission closure, approval, or permission token.

## Public/private boundary

Public examples must be generic and license-safe.

Private Village memory must stay local and ignored by Git.
