# Private literature extraction scroll

This Scroll defines how local private literature may be used to extract safe, reusable learning.

## Core rule

Private literature is a source of evidence, not public doctrine.

## Status

Draft.

## Activation

Use this Scroll when a Local Village stores private books, papers, notes, paid content, proprietary documentation, or converted files for learning purposes.

## Default mode

Read-only and local-only.

The agent may not copy private literature into public files, prompts, reports, commits, or shared outputs unless explicitly approved and license-safe.

## Allowed outputs

Allowed local outputs may include:

- source cards;
- personal notes;
- short extracted principles;
- review rubrics;
- learning proposals;
- local doctrine drafts.

Public outputs may include only:

- distilled principles;
- high level references;
- user-approved rules;
- license-safe summaries;
- original examples written by the user or agent.

## Forbidden outputs

Do not publish or commit:

- full book chapters;
- large copied excerpts;
- paid content;
- proprietary material;
- converted copyrighted source files;
- screenshots of protected material;
- private annotations containing sensitive data.

## Source card

Each source should have a card:

```text
Title:
Author:
Edition or version:
Source type:
License or access:
Local path:
Allowed use:
Forbidden use:
Extraction date:
Notes:
```

## Extraction workflow

1. Confirm the source is local and private.
2. Create or update a source card.
3. Extract short principles in original wording.
4. Avoid copying long passages.
5. Map each principle to a review rubric item.
6. Mark whether the principle is local-only or safe to promote.
7. Create a learning proposal if promotion is useful.
8. Wait for user approval before doctrine changes.

## Review rubric promotion

A principle can move into a rubric when it is:

- general enough to apply across missions;
- written in original language;
- not a copied passage;
- supported by more than one source or mission when possible;
- approved by the user.

## Doctrine promotion

A principle can become doctrine only after:

- Shikamaru drafts it;
- Jounin reviews risk and consistency;
- the user approves it;
- the change is committed intentionally.

## Report

The extraction report should include:

```text
Sources reviewed:
Local-only material:
Distilled principles:
Proposed rubric updates:
Proposed doctrine updates:
Copyright or sensitivity risks:
Approval needed:
```

## Completion

The mission is complete only when no private content leaked and the user can explain what was extracted, what stayed local, and what is proposed for promotion.
