# Security policy

Konoha Agentic Academy is local-first and safety-first.

The public repository must not contain secrets, credentials, private context, local Allied Village files, private literature, personal data, internal project data, or private assets.

## Reporting security issues

Do not open a public issue with:

- secrets;
- credentials;
- tokens;
- private keys;
- personal data;
- private project context;
- vulnerability details that could be abused.

Use GitHub private vulnerability reporting if it is enabled for this repository. If it is not enabled, contact the repository owner through a private channel.

## Public/private boundary

Before reporting or contributing anything related to local context, read:

- `docs/guides/public_private_boundary.md`
- `protocols/safety/safety_policy.md`
- `scrolls/sensitive_data_review_scroll.md`
- `scrolls/publication_safety_scroll.md`

## Supported versions

This project is early stage. Security handling applies to the current `main` branch unless a release policy says otherwise.

## Agent behavior

Agents must not access, summarize, copy, publish, or store sensitive content unless the action is explicitly allowed by an approved Mission Charter and the relevant safety rules.

See:

- `AGENTS.md`
- `core/conduct/AGENT_CONDUCT.md`
- `protocols/safety/safety_policy.md`
